## Architectural Gap Analysis

After analyzing all 300 queries, **11 distinct architectural gaps** emerged that aren't covered by the existing architecture. Each maps to a specific cluster of queries:

### GAP 1 — Effort estimation (queries 16-30, ~15 queries)
**Problem**: We have `difficulty` (skill required) but not `effort` (time required). A docs typo is `easy + 15min`. An auth refactor is `medium + 1day`.
**Fix**: Add `estimated_minutes` column, derived during sync via heuristics:
- `title contains "typo"|"fix link"` → 15min
- `issue_type='documentation' AND body_length<500` → 30min
- `body_length<200 AND has_code_block` → 1hr
- `body_length<1000` → 2-4hrs
- Otherwise → 1day+
- Add tool param `max_minutes` for "30 min fix" queries

### GAP 2 — Vague topic handling (queries 1-15, ~15 queries)
**Problem**: "find me something" → embed("find me something") returns garbage.
**Fix**: Detect vague topics in tool. If topic is generic ("anything", "something", "any issue"), **skip semantic search entirely** and return top issues by composite score with current filters. No embedding needed.

```typescript
const VAGUE_PATTERNS = /^(anything|something|any|whatever|surprise|find me|give me)/i;
if (VAGUE_PATTERNS.test(topic)) {
  // Skip embedding — order purely by composite score
  query_embedding = null;
}
```

### GAP 3 — Conversation result memory (queries 96-115, ~20 queries)
**Problem**: "the third one", "more like that", "skip what you showed" needs memory of previous results.
**Fix**: After each search, store result issue IDs in conversation context (existing memory system). Tool gets `previous_issue_ids` param to:
- Resolve "the third one" → issue #N
- Apply `exclude_issue_ids` for "show me different ones"

### GAP 4 — `similar_to_issue` parameter (queries 97, 108, 121)
**Problem**: "more like the second one", "issues like the ones I fixed before" — need similarity to a known issue.
**Fix**: 
```typescript
similar_to_issue_id: z.string().optional()
  .describe("Find issues similar to this one. Pass the issue's UUID. Uses its stored embedding.")
```
Implementation: fetch the source issue's embedding from DB, use it as query_embedding instead of embedding a topic string.

### GAP 5 — File path extraction (queries 174, 178, 183)
**Problem**: "show me the code area", "what files do I need to change?"
**Fix**: During sync, regex-extract file paths from body:
```typescript
const FILE_PATH_REGEX = /[\w\-\/]+\.(ts|tsx|js|jsx|py|rs|go|java|md|yml|yaml|json)/g;
const mentionedFiles = [...body.matchAll(FILE_PATH_REGEX)].map(m => m[0]).slice(0, 5);
```
Store as `mentioned_files text[]`. Show in results: "Likely files: src/auth.ts, middleware.ts"

### GAP 6 — Onboarding flow (queries 136, 141, 184-185)
**Problem**: "i'm completely new", "where do I start?", "step by step please"
**Fix**: Add `action: "onboard"` that returns a curated 5-step flow:
1. Pick a beginner-friendly repo (highest health_percentage + has_contributing)
2. Read its CONTRIBUTING.md
3. Pick a `difficulty=easy + has_help_wanted + well_described` issue
4. Comment to claim
5. Offer Daytona workspace setup

### GAP 7 — Persona detection (queries 132-135, 137-150)
**Problem**: "I'm a frontend dev", "I'm a backend python dev" → infer filters.
**Fix**: Add to system prompt — persona → filter mapping:
- `frontend` → languages: ["TypeScript", "JavaScript"], topics: ["react", "tailwind", "ui"]
- `backend` → languages: ["Python", "Go", "Rust"], topics: ["api", "database", "auth"]
- `devops` → topics: ["docker", "kubernetes", "ci-cd"]
- `designer` → issue_type: "documentation", body has_screenshots
- `writer` → issue_type: "documentation"

### GAP 8 — Aggregate stats action (queries 286-295)
**Problem**: "how many python issues?", "language distribution"
**Fix**: Add `action: "stats"` with `group_by` param:
```typescript
group_by: z.enum(["language", "difficulty", "issue_type", "repo", "labels"]).optional()
```
Returns aggregated counts via SQL `GROUP BY`.

### GAP 9 — Hour-level time precision (queries 82, 89)
**Problem**: "issues from the last hour" — `max_age_days: 1` is the smallest unit currently.
**Fix**: Add `max_age_hours` param. If set, takes precedence over `max_age_days`:
```sql
AND osi.updated_at > NOW() - (filter_max_age_hours || ' hours')::interval
```

### GAP 10 — Negative ID exclusion + result counts (queries 70, 72, 105, 110)
**Problem**: "exclude the cal.com one", "skip the assigned ones from these"
**Fix**: Already partially handled by filters, but add `exclude_repo_names` array param for explicit exclusion.

### GAP 11 — Real-time issue refresh (queries 186-200, ~15 queries)
**Problem**: Synced data is up to 6h stale. "Did anyone claim it yet?" needs LIVE data.
**Fix**: Add `action: "check_issue"` that bypasses DB and calls GitHub directly:
```typescript
GET /repos/{owner}/{repo}/issues/{number}
GET /repos/{owner}/{repo}/issues/{number}/timeline (last 5 events)
```
Returns live state, assignees, recent comments, linked PRs.
